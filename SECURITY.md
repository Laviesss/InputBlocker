# Security Policy

## Reporting a Vulnerability

We take the security of InputBlocker seriously. If you discover a security vulnerability, please do not open a public issue. Instead, report it privately to the maintainers.

**Contact**: Please reach out via the project's official communication channels or email the maintainer directly.

## Supported Versions

| Version | Supported | Notes |
|---|---|---|
| 0.x.x | ✅ | Current development branch. |

## Disclosure Process
1. **Reporting**: Submit a detailed report including the vulnerability type, steps to reproduce, and potential impact.
2. **Verification**: The maintainer will verify the issue.
3. **Patching**: A fix will be developed and tested.
4. **Disclosure**: Once the fix is released, the vulnerability may be disclosed publicly.
